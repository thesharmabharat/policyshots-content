# Policy Shots — Content

Articles for the Policy Shots app. Written as markdown; the app pulls them live
over HTTPS from this repo. No backend, no rebuild — push an article and it shows
up in the app.

## How to publish a new article

1. Create a new file in `articles/`, named `ps_NNN.md` (the filename becomes the
   article's `id` — keep them unique, e.g. `ps_025.md`).
2. Add the frontmatter block + write the body in markdown (template below).
3. Commit and push (or use GitHub's web editor → "Commit changes").
4. A GitHub Action regenerates `index.json` automatically (~20–30s).
5. Open the app → pull to refresh → your article appears.

## Article template

```markdown
---
title: Your headline here
excerpt: One-line summary shown on the card.
category: policy            # policy | business | economy | startup | capitalMarkets
type: daily                 # daily | markets | infographic
symbol: lock.shield.fill    # any SF Symbol name (thumbnail icon)
publishedAt: 2026-06-23T09:00:00Z
readMinutes: 4              # OPTIONAL — auto-computed from word count if omitted
image: images/ps_025.jpg    # OPTIONAL — hero image path
---

## A subheading

Your article body in **markdown**. Headings, lists, *italics*, [links](https://example.com),
> blockquotes — all supported.
```

### Field reference

| Field         | Required | Notes |
|---------------|----------|-------|
| `title`       | yes      | Headline. |
| `excerpt`     | yes      | Short summary for the card. |
| `category`    | yes      | One of: `policy`, `business`, `economy`, `startup`, `capitalMarkets`. |
| `type`        | yes      | One of: `daily`, `markets`, `infographic`. |
| `publishedAt` | yes      | ISO 8601 UTC, e.g. `2026-06-23T09:00:00Z`. Controls sort order. |
| `symbol`      | no       | SF Symbol name for the thumbnail. |
| `readMinutes` | no       | Auto-computed from word count (~200 wpm) if omitted. |
| `image`       | no       | Path to a hero image in `images/`. |

> **Don't hand-edit `index.json`** — it's generated by the GitHub Action from the
> article frontmatter on every push.

## Repo layout

```
articles/        one .md per article (filename = id)
images/          optional article images
index.json       AUTO-GENERATED manifest (the app reads this first)
scripts/         build_index.py — generates the manifest
.github/         the Action that runs it
```

## Regenerating the index locally (optional)

```bash
python3 scripts/build_index.py
```
